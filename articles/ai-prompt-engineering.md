# Prompt 工程最佳实践

同一个模型，好的 prompt 和差的 prompt 效果天差地别 📝

## 基础原则

1. **清晰明确**：把任务说清楚，别让模型猜
2. **结构化**：用分隔符、编号让 prompt 有层次
3. **给例子**：Few-shot 示例效果拔群

## 常用技巧

### Zero-shot（不给例子）
```
把这段话翻译成英文：
```

### Few-shot（给例子）
```
把中文翻译成英文：
苹果 -> apple
香蕉 -> banana
老师 ->
```

### Chain-of-Thought（思维链）
```
这个问题分步骤思考：
1. 先理解问题
2. 分析条件
3. 计算
4. 给出答案
问题：15 + 27 = ?
```

### 角色扮演
```
你是一个资深工程师，擅长代码审查。请指出下面代码的问题：
```

## Prompt 模板

```markdown
## 角色
你是一个 [角色]

## 任务
[具体任务描述]

## 要求
- 要求1
- 要求2

## 示例
[给1-2个例子]

## 输出格式
[JSON/Markdown/纯文本]
```

## 避坑

- ❌ 模糊不清："帮我写点东西"
- ❌ 一次问太多："顺便帮我检查语法，再翻译，再总结"
- ❌ 不设限制："随便写" → 可能放飞

> 多练多用，prompt 也是一门艺术 🎨
